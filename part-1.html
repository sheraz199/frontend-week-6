Scoping Your First Major App

Before writing a single line of code, professional engineers plan. In this lesson, you will learn the industry standards for defining project scope, establishing an MVP, and planning the folder structure for a multi-page web application.

1. Defining the "Scope"

In software engineering, Scope is the strict boundary of what your application will—and will not—do.

Junior developers often start coding immediately and continuously add random features. This leads to Feature Creep, resulting in delayed, buggy, and unfinished projects.

The MVP Rule: Always build the Minimum Viable Product (MVP) first. If you are building a social network, the MVP is the ability to post a message and see other messages. Direct messaging and custom profile themes are outside the MVP scope and must wait for Version 2.

2. Mapping the Pages (Wireframing)

For a multi-page application (like an E-Commerce site or Student Portal), you must map out the user journey. What screens exist, and how do users navigate between them?

  • /Home: The landing page with featured products.
  • /Shop: A grid of all available products with filters.
  • /Cart: The checkout review page.
  • /Profile: User settings and order history.

By listing these out, we immediately know we need a Global Navbar component that will exist across all these views so the user can easily navigate.

3. Architecture & Folder Structure

In modern frontend frameworks (React, Vue, etc.), we don't dump 50 files into one folder. We organize our code cleanly based on its purpose.

Industry Standard React Structure 
/src
  /assets         # Images, fonts, and global CSS
  /components     # Reusable UI (Buttons, Navbars, Cards)
  /pages          # Full view screens (Home.jsx, Cart.jsx)
  /context        # Global State logic (User Login Data)
  App.jsx         # The main file handling URL routing
  main.jsx        # Injects the app into the DOM

Notice the distinction: A page represents a whole screen, while a component is just a small piece of a screen.

Why this is essential for your career:

  1. Agile Teamwork: In a professional setting, a project manager will hand you a strict "scope document" (often via Jira). You must know how to read these requirements and translate them into a component tree.
  2. Time Management: Planning your state and folders ahead of time prevents "spaghetti code," saving you hours of rewriting code when the app gets larger.
  3. Scalability: Following standard folder structures means that if a new developer joins your team tomorrow, they immediately know where to find the Navbar and where to find the Home Page.
Action Item: Before our next lesson, sketch your app's pages on a piece of paper. Draw boxes for the components you think you will need (Navbar, Sidebar, Footer, Data Cards).

Planning Complete!
