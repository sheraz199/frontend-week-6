Building the Core Structure

With our project scoped and planned, it is time to lay the foundation. We will learn how to set up routing for a multi-page app, create a global layout, and build the core base components.

1. React Router: The Traffic Cop

React builds "Single Page Applications" (SPAs). This means there is only one actual HTML file. So, how do we have a /home page and a /cart page?

We use an industry-standard library called React Router. It acts like a traffic cop: it watches the URL in the browser, and based on the URL, it swaps out which Components are visible on the screen.

App.jsx (The Entry Point) 
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Home from './pages/Home';
import Cart from './pages/Cart';
import Layout from './components/Layout';

export default function App() {
  return (
    <BrowserRouter>
      <Routes>
        {/* The Layout wraps around all our pages */}
        <Route path="/" element={<Layout />}>
          <Route index element={<Home />} />
          <Route path="cart" element={<Cart />} />
        </Route>
      </Routes>
    </BrowserRouter>
  );
}

2. The Global Layout & Outlet

In older web development, you had to copy and paste your Navbar and Footer code into every single HTML page. In React, we create a Layout Component.

The Layout contains the Navbar at the top, the Footer at the bottom, and a special React Router component called <Outlet /> in the middle. The <Outlet /> acts as a dynamic window where the page content is injected.

Layout.jsx 
import { Outlet } from 'react-router-dom';
import Navbar from './Navbar';
import Footer from './Footer';

export default function Layout() {
  return (
    <div className="app-wrapper">
      <Navbar />
      
      {/* The Outlet is where Home.jsx or Cart.jsx will render */}
      <main className="main-content">
        <Outlet />
      </main>

      <Footer />
    </div>
  );
}

3. Navigation without Refreshing

Standard HTML anchor tags (<a href="/cart">) force the browser to refresh the entire page. This destroys our application's State and makes the website feel slow.

Instead, we use the <Link> component provided by React Router. It updates the URL and changes the <Outlet /> content instantly, without refreshing the page.

Navbar.jsx 
import { Link } from 'react-router-dom';

export default function Navbar() {
  return (
    <nav className="global-nav">
      <h1>NexusShop</h1>
      <ul>
        <li><Link to="/">Home</Link></li>
        <li><Link to="/products">Shop</Link></li>
        <li><Link to="/cart">Cart</Link></li>
      </ul>
    </nav>
  );
}

Why this is essential for your career:

  1. The Core of SPAs: Understanding how routing works in a Single Page Application is a fundamental requirement for modern frontend development. Almost zero modern apps use hard page refreshes anymore.
  2. DRY Architecture: The <Layout /> pattern ensures you never repeat yourself. If you need to update the Navbar design, you change one file, and the update is instantly reflected across every page of your app.
  3. State Preservation: Because the page never actually reloads when using <Link> tags, your user's data (like items in their shopping cart or their login status) remains perfectly safe in your application's memory.
Common Beginner Mistake: Never use a standard <a href="..."> tag for internal links within your React application. Always import and use the <Link> component. Only use standard anchor tags when linking to external websites (like Google or Twitter).
